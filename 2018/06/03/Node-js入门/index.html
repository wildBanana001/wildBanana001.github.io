Nodejs

nodejs运行在服务器端，通过将V8引擎内嵌到操作系统的集成层，来让JavaScript享受到底层操作系统的异步接口，从而实现将其带到服务器端的目的。
nodejs 高效、简洁。无须将书写部署到独立安装的web服务器中去运行。

事件驱动和非阻塞模型

nodejs使用事件驱动和非阻塞模型。
事件驱动
I/O等异步操作结束后的通知
观察者模式
阻塞与非阻塞IO
阻塞：I/O时进程休眠等待I/O完成后进行下一步
非阻塞：I/O时函数立即返回，进程不等待I/O完成

Nodejs的单线程

单线程只是针对主进程，I/O操作系统底层多线程调度。
单线程并不是单进程。

共享状态的并发
Node是单线程的：故许多优秀的Node模块都是非阻塞的，执行任务采用异步方式。

事件轮询

事件轮询(http、net模块中的IO，timeout)
Node会先注册事件，随后不停地询问内核这些事件是否已经分发。当事件分发时，对应的回调函数就会被触发，然后继续执行。如无事件触发，则继续执行其他代码，直到有新事件时，再去执行对应的回调函数(异步操作)。

错误处理

错误处理
Node的执行环境是单线程的，应避免使用同步IO且所有的状态都是维护在一个内存空间中的，必须小心避免错误！一旦遗漏错误，将很难追踪，上下文信息会丢失。可能抛出的错误都要及时处理。

Node下的JavaScript

global对象 (全局对象GO)
在浏览器中全局对象指window对象。
Node中global和window一样，任何global对象上的属性都可以被全局访问。
process：所有全局执行上下文中的内容都在pricess对象中。Node中只有一个process对象等同于浏览器中的window。 例：window.name 等同 process.title。
模块系统：使用NPM安装模块，给模块独立定义专门的API。避免对全局命名空间的污染及命名冲突问题。
绝对模块：内部node_modules上可以查找到的模块，如http模块。
相对模块：require指向一个相对工作目录中的JS文件。
暴露API：使用exports打包

require：支持js、json、node扩展名，不写依次尝试。不写路径则默认是build-in模块或者各级node_modules内的第三方模块。

Buffer

Buffer类(缓冲)是Nodejs的全局变量，因此无需使用require(‘buffer’).Buffer
用于处理二进制数据流，实例类似整数数组，大小固定。Buffer使用内存是C++代码在V8堆外分配的物理内存。
ES6之前JS语言没有读取或操作二进制数据流的机制，Buffer类被引入作为Nodejs的API的一部分，使TCP流或文件系统操作中处理二进制数据流。
如今ES6有TypedArray，比Buffer类更优化、更适合Nodejs。

事件

Event 事件
监听器，所有能触发事件的对象都是EventEmitter类的实例，这些对象开发了一个eventEmitter.on()函数 (注册监听器)，可以将一个或多个函数绑定到会被对象触发的命名事件上。eventEmitter.emit() （触发事件）

fs

fs(文件系统)
文件I/O是由简单封装的标准POSIX函数提供。通过require(‘fs’)使用该模块，所有方法都有同步异步的形式。

MongoDB

MongoDB是一个面向文档，无模式(关系）的数据库，非常适合于Node.js应用以及云端部署。
可以将任意类型的文档数据存储到集合中。
可以将数据看作文档，设计非常灵活。绝大多数情况下就是JSON格式形式存储。
可以与其他键–值形式的NoSQL数据区别开，文档可以是任意深度的。
