javascript的prototype及继承

原型（prototype） & 原型链

原型定义: 原型是 function 对象的一个属性，它是构造函数制造出来的对象的公共祖先。通过这个构造函数产生的对象，可以继承该原型的属性和方法。原型也是对象。

JS中的__proto__总是指向prototype。__proto__并不是JS语言本身的特性，而是各大厂商具体实现时添加的私有属性，但不推荐使用，避免对环境产生依赖。

构造函数可以通过function.prototype = {};来 添加 / 修改 原型上的属性
对象可以通过object.__proto__ = {};来 添加 / 修改 原型上的属性

注意：undefined，null 没有原型

原型链定义: 原型的对象还有自己的原型，这样原型上还有原型的结构就构成了原型链。

继承

继承（6种方法）：

  1. 原型链继承

缺点：会继承过多没有用的属性，造成大量的浪费。

  1. 构造函数继承 call / apply

缺点：这种方式不属于继承，也访问不了原型的原型。每次构造一个对象都要走一个构造函数，效率很低。

  1. 共享原型

缺点：没办法改变子类的原型，一改就两个一起改了。

  1. object.create() 在原型上生成新的对象的实例

缺点：object.creat方法生成的新对象，动态继承了原型。在原型上添加或修改任何方法，会立刻反映在新对象上。所以这个继承得到的不是类，只是得到对象。
好处：这个方法在你想要改变一个的原型来创建实例而言会特别好用。

  1. 圣杯模式

F 函数的作用：沟通 P 和 C 的原型，这样我们改变 C 的原型的时候只会影响 F 而不会影响 P。

  1. extands（es6 方法）

顺便我们在说说我们在改变原型时使用了new这个关键字。

问：new是什么？
答：new其实是JS语法中的一个语法糖。当我们要创建一个对象的实例时。我们要去继承这个对象的原型。那么new的作用就是帮你自动完成4个步骤。
①帮你创建一个新对象。var obj = new Object();
②帮你绑定原型。obj.__proto__ = Object.prototype 将新对象的__proto__属性指向构造函数的prototype
③帮你将this指向新对象。
④帮你return新对象。但在这里注意构造函数是否有返回值。如果构造函数的返回值为基本数据类型（number，string，boolean，undefined，null）则返回临新对象。如果构造函数的返回值为对象类型，则返回这个对象类型。

那么我们在说说this是什么？

this是一个指向。在函数预编译过程中this指向window对象。在全局作用域里this也指向window对象。
最后this永远指向最后调用它的那个对象。
那么我们如何改变this指向？
①在函数内部使用 var that = this
②使用apply、call、bind函数改变this指向
③用new实例化一个对象
④使用es6的箭头函数

其中call/apply作用都是改变this指向，区别只是后面传的参数形式不同。call(thisArg,[arg1,[arg2,[...]]])，apply(thisArg,[arrs])
bind也能改变this指向。但是bind()方法会创建一个新函数，称为绑定函数，当调用这个绑定函数时，绑定函数会以创建它时传入bind()方法的第一个参数作为this，传入bind方法的第二个以及以后的参数加上绑定函数运行时本身的参数按照顺序作为原函数的参数来调用原函数。
